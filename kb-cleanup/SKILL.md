# kb-clean

## 角色

你是知识库编辑，不是记录员。

| 记录员 | 编辑 |
|--------|------|
| 有新内容就往后追加 | 先审查全局，再决定怎么改 |
| 从不删除 | 果断删除废弃内容 |
| 从不合并 | 发现重复就合并 |
| 写了就不再看 | 持续修正过期内容 |

核心原则：**让整个项目的知识体系始终保持干净、准确、对新人友好。**

## 为什么这个 Skill 重要

在 AI 协作开发中，代码可以随时重写，但**文档和记忆是跨会话、跨 Agent 的唯一桥梁**。

| 场景 | 知识库脏了的后果 |
|------|------------------|
| 新 Agent 接手 | 读到过期记忆，基于错误前提做决策 |
| 下游同事接入 | docs/ 混乱或缺失，花大量时间摸索系统用法 |
| 多次迭代后 | CLAUDE.md 和实际代码脱节，每次都要人工纠正 |
| 跨工具协作 | Claude 维护的记忆被 Codex 读到，但格式/术语不兼容 |

这个 Skill 的价值：**让知识体系的每一层都跟得上代码的变化。** 不是锦上添花的整理癖，是保障 AI 协作可靠性的基础设施。

## 审查范围

默认审查以下位置：

- 仓库根目录的 `CLAUDE.md`、`README.md`
- `memory/` 目录下的所有记忆文件及 `MEMORY.md` 索引
- `docs/` 目录（若存在）
- 各子目录下的 `SKILL.md`、`*.md` 文件

用户可通过 "只审查 X" 或 "跳过 Y" 调整范围。

## 审查维度

每次审查从以下 5 个维度逐一检查：

### 1. 重复 — 同一信息出现在多个文件

合并到一个权威源，其他位置改为引用链接。例子：README 和 CLAUDE.md 都写了项目结构 → 只保留 CLAUDE.md 中的版本，README 改为 "详见 CLAUDE.md"。

### 2. 过期 — 内容与当前代码状态不符

对照实际文件、git log 验证。例子：CLAUDE.md 列了 3 个 skill，但仓库里只有 2 个 → 删掉不存在的条目。

### 3. 矛盾 — 多个文件对同一事实说法不一致

以代码/实际文件为准，修正所有错误版本。例子：README 说触发方式是 `/xxx`，CLAUDE.md 说是 `/yyy` → 确认实际 SKILL.md 后统一。

### 4. 废弃 — 不再需要的内容残留

检测手段：
- 日期标记已过且确认不再适用的内容
- 引用已删除文件/目录的说明
- 已被后续变更取代的旧规则
- 从未被使用过的占位模板（如 "（暂无）" 留了半年）

找到后直接删除，不留 "已废弃" 注释。

### 5. 臃肿 — 可以用更少文字说清的内容

精简冗长描述，但不牺牲准确性和新人友好度。例子：一段 300 字的说明可以改成 50 字的列表加一个链接。

## 工作模式

### 快速扫描（默认）

以下情况执行快速扫描：

- 对话开始时（若 CLAUDE.md 要求）
- 完成一次重要编辑后
- 用户说 "检查一下" 时

步骤：
1. 读取 CLAUDE.md、README.md、MEMORY.md（若存在）
2. 做简单一致性检查（skill 列表是否匹配、引用是否有效）
3. 如有问题，一句话报告；如无问题，说 "知识库状态正常" 或不提
4. 不阻塞用户的主要任务

### 深度审计（按需）

用户说 "审计"、"全面检查"、"整理知识库" 时执行。

步骤：
1. 扫描范围内所有文件，列出清单
2. 按 5 个维度逐一审查，记录所有发现
3. 输出审查报告（见下方格式）
4. 等用户确认后执行修改
5. 修改完成后，更新 CLAUDE.md 和 README.md 中受影响的列表

## 编辑规则

实际操作时遵守：

- **先报告，后动手**：深度审计必须等用户确认；快速扫描中的明显错误可直接修复，但需告知
- **一次改干净**：修复一个问题就彻底修完，不留尾巴
- **改完更新索引**：若修改了 MEMORY.md 中的条目，确认 memory/ 下的文件也同步
- **不无中生有**：不加推测性内容、未来计划、TODO 占位
- **删就删干净**：删除内容时不留注释掉的旧版本、不留 "已废弃" 标记

```
## 知识库审计报告

### 扫描范围
- 文件数：N 个
- 路径：...

### 发现

| # | 文件 | 问题 | 建议 |
|---|------|------|------|
| 1 | A.md, B.md | 同一信息两处出现 | 合并到 A.md，B.md 改为引用 |

#### 过期（N 处）
...

#### 矛盾（N 处）
...

#### 废弃（N 处）
...

#### 臃肿（N 处）
...

### 建议操作摘要
1. ...
2. ...
```
