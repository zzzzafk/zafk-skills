# zafk-skills

专门维护 Claude Code skills 的仓库。每个 skill 是独立的功能模块，封装特定的工作流程或领域知识。

## 目录结构

```
zafk-skills/
├── CLAUDE.md          # 详细说明文档（本文件）
├── README.md          # 项目简介
├── api-generator/     # API文档解析与代码生成
│   └── SKILL.md
├── kb-clean/          # 知识库审查清理
│   └── SKILL.md
├── ts-code-quality/   # TypeScript/Vue代码质量规范
│   └── SKILL.md
├── webapp-testing/    # Web应用测试工具
│   ├── examples/
│   ├── scripts/
│   └── SKILL.md
└── memory/            # 仓库级记忆系统
```

## 设计原则

每个 skill 应遵循以下原则：

- **独立性**：每个 skill 应能独立使用，不依赖其他 skill
- **封装性**：skill 内部结构和外部依赖说明清楚
- **渐进披露**：SKILL.md 名称+描述即可触发，正文按需加载
- **可验证**：每个 skill 应有明确的测试方式和预期效果

## 角色定位

你是知识库编辑，不是记录员。审查全局、合并重复、修正过期、删除废弃。让知识体系保持干净、准确、对新人友好。

## 技能详解

### 1. api-generator

**功能**：自动读取远程 API 文档(URL)或本地文件，解析并生成 TypeScript/JavaScript 类型定义和 API 请求函数。支持 OpenAPI 3.x / Swagger 2.x 格式。

**适用场景**：
- 后端提供了新的 API 文档，需要生成前端类型和请求函数
- 后端接口变更，需要同步更新前端代码
- 创建新项目时快速生成接口层代码

**触发方式**：
- 自然语言："同步接口"、"生成API类型"、"写API请求"、"更新接口定义"、"拉取文档"、"生成接口代码"、"API文档转代码"
- 命令：`/api-generator`

**依赖**：无

**输入/输出**：
- 输入：API 文档的 URL 或本地文件路径，可选输出格式（TS/JS）
- 输出：生成的类型定义文件（types.ts/types.js）和请求函数文件（api.ts/api.js）

---

### 2. kb-clean

**功能**：审查和清理项目知识文件——CLAUDE.md、README.md、memory/ 及所有 markdown 文档。从重复、过期、矛盾、废弃、臃肿五个维度审计。

**适用场景**：
- 对话开始时自动检查知识库状态
- 新增或修改 skill 后确保文档一致性
- 用户要求审计、整理、清理文档时

**触发方式**：
- 自然语言："检查知识库"、"审计文档"、"整理一下"
- 自动：对话开始时自动执行快速扫描

**依赖**：无

**输入/输出**：
- 输入：无（读取项目文件）
- 输出：审计报告（深度审计时）或简短状态确认（快速扫描时）

---

### 3. ts-code-quality

**功能**：TypeScript/Vue 3 代码质量规范检查，强制零警告容忍、组件大小限制、代码复用等规则。自动检测项目技术栈（TS支持、Vue版本）并应用相应规则。

**适用场景**：
- 编写或修改 Vue/TypeScript 代码时进行规范检查
- Code Review 时验证代码质量
- 确保项目代码风格一致性

**触发方式**：
- 自然语言："代码审查"、"代码规范检查"、"Vue组件规范"、"TypeScript类型检查"
- 自动：编写代码时自动应用

**依赖**：项目需安装 ESLint、TypeScript（如适用）

**输入/输出**：
- 输入：待检查的代码文件或代码片段
- 输出：规范检查报告，指出不符合规则的问题及修复建议

---

### 4. webapp-testing

**功能**：使用 Playwright 测试本地 Web 应用，支持功能验证、UI 调试、截图和日志捕获。提供服务器生命周期管理工具。

**适用场景**：
- 测试本地开发服务器上的 Web 应用
- 自动化 UI 测试
- 捕获浏览器日志进行调试

**触发方式**：
- 自然语言："测试网页"、"自动化测试"、"浏览器自动化"、"UI测试"
- 命令：`/webapp-testing`

**依赖**：Python 3.x、Playwright 库

**输入/输出**：
- 输入：服务器启动命令、端口号、测试脚本或测试需求描述
- 输出：测试结果报告、截图、日志文件
